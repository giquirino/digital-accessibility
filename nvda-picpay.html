O que o NVDA detecta que nenhuma ferramenta automatizada detecta

Abaixo: transcrição do que o NVDA anuncia em voz alta + o código responsável + a correção.

1

Botões anunciados como "button" sem contexto

WCAG 4.1.2 — Name, Role, Value · Impacto: o usuário não sabe o que o botão faz
🔊 O que o NVDA anuncia:
"button"
"button"
"button"
❌ Código que causa isso <button class="teaser__dot active" data-gtm-button-decorated="true"> </button> <!-- NVDA lê: "button" — sem nenhum contexto -->
🔊 Após correção, NVDA anuncia:
"Ir para o slide 1 de 5, button"
✅ Correção <button class="teaser__dot active" aria-label="Ir para o slide 1 de 5"> </button>
2

Links anunciados como "link" sem destino

WCAG 2.4.4 — Link Purpose · O usuário não sabe para onde o link vai
🔊 O que o NVDA anuncia ao navegar com Tab:
"link"
"link"
"link"
❌ Código que causa isso <a href="https://blog.picpay.com/carteira-de-trabalho-digital/" role="button" data-gtm-name="CARTEIRA_DE_TRABALHO_DIGITAL"> </a>
🔊 Após correção:
"Ler artigo: Carteira de trabalho digital, link"
✅ Correção <a href="https://blog.picpay.com/carteira-de-trabalho-digital/" aria-label="Ler artigo: Carteira de trabalho digital"> </a>
3

Navegação por headings com saltos (tecla H)

WCAG 2.4.6 — Headings and Labels · Sumário de página incompleto
🔊 NVDA ao pressionar H repetidamente:
"heading level 1: PicPay"
"heading level 2: Conheça nossos cartões"
"heading level 4: PicPay Card" ← pula nível 3!
"heading level 4: PicPay Gold" ← pula nível 3!
🔊 NVDA ao pressionar tecla "3" (ir para h3):
"no heading level 3"
❌ Estrutura do picpay.com <h1>PicPay</h1> <h2>Conheça nossos cartões</h2> <h4>PicPay Card</h4> <!-- sem h3 -->
✅ Correção <h1>PicPay</h1> <h2>Conheça nossos cartões</h2> <h3>PicPay Card</h3>
4

Ausência de <main> — landmark navigation quebrada

Best Practice · Usuário não consegue pular direto pro conteúdo
🔊 NVDA ao pressionar D (próximo landmark):
"banner" (header)
"content info" (footer) ← pula tudo!
Impacto real: sem <main>, o usuário precisa percorrer TODOS os links do menu (dezenas) antes de chegar ao conteúdo. Com <main>, bastaria pressionar D uma vez.
❌ Estrutura do picpay.com <body> <div class="header">...</div> <div class="content">...</div> <!-- NVDA ignora --> <div class="footer">...</div> </body>
✅ Correção — landmarks semânticos <body> <header>...</header> <main>...</main> <!-- NVDA: tecla D pula pra cá --> <footer>...</footer> </body>
5

Imagens sem alt — NVDA lê a URL em voz alta

WCAG 1.1.1 — Non-text Content · URL incompreensível lida em voz alta
🔊 O que o NVDA anuncia:
"image, publish-p153277-e1666050 adobeaemcloud dot com adobe dynamicmedia deliver dm-aid..."
❌ Código que causa isso <img src="https://publish-p153277-e1666050.adobeaemcloud.com/..." loading="eager" decoding="async" width="200" height="250">
🔊 Após correção:
"Cartão PicPay na variante verde, image"
✅ Correção <img src="..." alt="Cartão PicPay na variante verde" loading="eager">
6

Contraste — invisível no modo alto contraste

WCAG 1.4.3 — Contrast · Texto desaparece com High Contrast Mode ativado
❌ CSS do PicPay — invisível com NVDA High Contrast .wrapped-teaser h2 { color: #999999; /* desaparece no modo alto contraste */ }
✅ Correção com suporte a forced-colors .wrapped-teaser h2 { color: #595959; } /* Bônus — respeita Windows High Contrast Mode */ @media (forced-colors: active) { .wrapped-teaser h2 { color: CanvasText; } }
7

Ordem de leitura ≠ ordem visual (CSS order)

WCAG 1.3.2 — Meaningful Sequence · Conteúdo lido em ordem errada
🔊 Quem enxerga vê: Card 2 → Card 3 → Card 1
"PicPay Card" (Card 1)
"PicPay Gold" (Card 2)
"PicPay Platinum" (Card 3)
❌ CSS reorganiza visualmente mas DOM está em outra ordem .cardgrid__item:nth-child(1) { order: 3; } .cardgrid__item:nth-child(2) { order: 1; } .cardgrid__item:nth-child(3) { order: 2; }
✅ Correção — reordenar no DOM, não no CSS <!-- Ordem do HTML = ordem de leitura desejada --> <div class="cardgrid__item">Card 2</div> <div class="cardgrid__item">Card 3</div> <div class="cardgrid__item">Card 1</div>
Exclusivo do NVDA: nenhuma ferramenta automatizada detecta discrepância entre ordem visual (CSS) e ordem do DOM. Só um leitor de tela real revela isso.
