Integração CI/CD

Quality gate automatizado com GitHub Actions

O axe-core CLI é feito para pipelines — o flag --exit faz o job falhar se encontrar violations, bloqueando o merge automaticamente.

📄 .github/workflows/accessibility.yml name: Accessibility Gate on: [pull_request] jobs: a11y-check: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Install axe-core CLI run: npm install -g @axe-core/cli - name: Run accessibility audit run: | axe https://staging.picpay.com/ \ --exit \ --reporter json \ --stdout > axe-results.json # --exit = job FALHA se tiver violations - name: Upload evidence if: always() uses: actions/upload-artifact@v4 with: name: axe-report path: axe-results.json
Na prática: PR #142 — "feat: novo banner homepage" → Job a11y-check ❌ FAILED → axe found 2 critical violations (button-name, link-name) → PR bloqueada até corrigir.
