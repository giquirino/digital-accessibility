Erros Encontrados

Problemas que precisam de correção

O WAVE marca ícones coloridos diretamente sobre os elementos na página. Abaixo estão os erros com exemplos de código reais do picpay.com.

ERROR

Empty button — Botões sem nome acessível

WCAG 4.1.2 — Name, Role, Value · Tipo WAVE: Empty button
3 elementos afetados — dots do carrossel principal
❌ Encontrado no picpay.com <!-- WAVE marca com ícone vermelho ❌ sobre o botão --> <button class="teaser__dot active" data-gtm-button-decorated="true"> </button>
✅ Correção <button class="teaser__dot active" aria-label="Ir para o slide 1 de 5"> </button>
Por que o WAVE detecta: verifica se o <button> possui algum conteúdo textual (texto visível, aria-label, aria-labelledby ou title). Se está vazio, marca como "Empty button".
ERROR

Missing alt text — Imagens sem alternativa

WCAG 1.1.1 — Non-text Content · Tipo WAVE: Missing alternative text
4 elementos afetados — imagens de cartões
❌ Encontrado no picpay.com <!-- WAVE mostra ícone vermelho posicionado sobre a imagem --> <img src="https://publish-p153277-e1666050.adobeaemcloud.com/..." loading="eager" decoding="async" width="200" height="250">
✅ Correção — imagem informativa <img src="https://publish-p153277-e1666050.adobeaemcloud.com/..." alt="Cartão PicPay na variante verde com chip dourado" loading="eager" decoding="async" width="200" height="250">
✅ Correção — imagem decorativa <img src="..." alt="" role="presentation" loading="eager">
Diferencial WAVE: mostra o ícone diretamente posicionado sobre a imagem na página renderizada — mais intuitivo que um JSON.
ERROR

Empty link — Links sem texto

WCAG 2.4.4 — Link Purpose · Tipo WAVE: Empty link
3 elementos afetados — links do carrossel de blog
❌ Encontrado no picpay.com <a href="https://blog.picpay.com/carteira-de-trabalho-digital/" role="button" data-gtm-name="CARTEIRA_DE_TRABALHO_DIGITAL"> </a>
✅ Correção — texto visível (preferencial) <a href="https://blog.picpay.com/carteira-de-trabalho-digital/"> Ler artigo: Carteira de trabalho digital </a>
CONTRAST

Very low contrast — Texto ilegível

WCAG 1.4.3 — Contrast (Minimum) · Ratio: 2.28:1 (mínimo 4.5:1)
2 elementos afetados — subtítulos de seção
❌ CSS encontrado — ratio 2.28:1 .wrapped-teaser h2 { color: #999999; /* cinza claro sobre fundo branco */ }
✅ Correção — ratio 7:1 .wrapped-teaser h2 { color: #595959; /* sobre fundo branco = 7:1 ✓ */ }
Painel WAVE de contraste: mostra a ratio exata de cada elemento + sugere cor alternativa que atinge o mínimo WCAG.
Alertas

Problemas que precisam de verificação

Alertas não são erros confirmados — exigem verificação humana pra determinar se afetam a acessibilidade.

ALERT

Skipped heading level — Hierarquia com saltos

WCAG 2.4.6 — Headings and Labels · 5 elementos afetados
❌ Encontrado no picpay.com <h1>PicPay — Tecnologia que simplifica</h1> <h2>Conheça nossos cartões</h2> <h4>PicPay Card</h4> <!-- ⚠️ pula h3 --> <h4>PicPay Gold</h4> <!-- ⚠️ pula h3 -->
✅ Correção — hierarquia sequencial <h1>PicPay — Tecnologia que simplifica</h1> <h2>Conheça nossos cartões</h2> <h3>PicPay Card</h3> <h3>PicPay Gold</h3>
ALERT

Very small link/button — Alvos de toque minúsculos

WCAG 2.5.5 — Target Size · 13 elementos afetados
❌ CSS encontrado — dots de 8×8px .cardgrid__dot { width: 8px; height: 8px; border-radius: 50%; padding: 0; }
✅ Correção — expandir área sem mudar visual .cardgrid__dot { width: 8px; height: 8px; border-radius: 50%; position: relative; } .cardgrid__dot::before { content: ''; position: absolute; inset: -8px; /* área efetiva: 24×24px */ }
ALERT

Accessible name mismatch — Label ≠ texto visível

WCAG 2.5.3 — Label in Name · 2 elementos afetados
❌ Encontrado — texto diz "Baixe o app" mas aria-label diz "Baixar o aplicativo" <a href="https://marketing.marketing.picpay.com/HPHx/tql2vlv1" role="button" aria-label="Baixar o aplicativo PicPay"> Baixe o app <!-- "Baixe" ≠ "Baixar" --> </a>
✅ Correção — aria-label contém o texto visível <a href="https://marketing.marketing.picpay.com/HPHx/tql2vlv1" role="button" aria-label="Baixe o app PicPay para Android ou iOS"> Baixe o app </a>
Por que importa: usuários de controle por voz (Dragon, Voice Control) dizem o texto que veem. Se o aria-label não começa com esse texto, o comando de voz falha.
